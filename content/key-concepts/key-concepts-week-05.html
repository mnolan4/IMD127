Week 5: Functions and Modularity - Key Concepts

Learning Outcomes

By the end of this week, you should be able to define your own functions. You should use parameters to make functions flexible. You should return values from functions. You should organize code into reusable modules.

Key Concepts

Custom Functions

Functions are blocks of code that perform a specific task. They let you organize your code, avoid repetition, and create reusable components. Once you define a function, you can call it multiple times from anywhere in your code.

Why Use Functions:

  • Avoid repetition: Write code once, use it many times
  • Organization: Break complex code into manageable pieces
  • Readability: Function names describe what the code does
  • Maintainability: Change code in one place, affects all uses
  • Reusability: Use the same function in different projects

Basic syntax:

Code example 1 written in javascript. The code follows.
// Define the function
function functionName() {
  // Code to run when function is called
  console.log("Function is running!");
}

// Call (invoke) the function
functionName();  // Runs the code inside the function
functionName();  // Can call it multiple times

Function Definition vs Function Call:

  • Definition: function myFunction() { ... } - Creates the function
  • Call: myFunction() - Actually runs the function
  • You must define a function before you can call it

Function Naming:

  • Use descriptive names that explain what the function does
  • Start with a verb: drawCircle(), moveObject(), calculateDistance()
  • Use camelCase: myFunctionName (not my_function_name)

Function Parameters

Parameters are variables that receive values when you call a function. They make functions flexible and reusable with different inputs.

How Parameters Work:

  1. Define parameters in the function definition: function drawCircle(x, y, size)
  2. Parameters act like variables inside the function
  3. When you call the function, you provide values (arguments): drawCircle(100, 100, 50)
  4. The values are assigned to the parameters in order
Code example 2 written in javascript. The code follows.
// Function with parameters
function drawCircle(x, y, size) {
  // x, y, and size are available inside the function
  ellipse(x, y, size, size);
}

// Call with different values each time
drawCircle(100, 100, 50);   // x=100, y=100, size=50
drawCircle(200, 200, 75);   // x=200, y=200, size=75
drawCircle(300, 300, 100);  // x=300, y=300, size=100

Parameter vs Argument:

  • Parameter: Variable in function definition (x, y, size)
  • Argument: Value passed when calling function (100, 100, 50)

Multiple Parameters:
Functions can have any number of parameters. Separate them with commas:

Code example 3 written in javascript. The code follows.
function drawShape(x, y, w, h, r, g, b) {
  fill(r, g, b);
  rect(x, y, w, h);
}

drawShape(100, 100, 200, 150, 255, 0, 0);  // Red rectangle

Benefits:

  • Flexibility: One function works with different values
  • Efficiency: Write once, use many times
  • Clarity: Function name + parameters describe what it does
  • Maintainability: Change function code once, all calls update

Return Values

Functions can calculate and return values that you can use elsewhere in your code. The return statement sends a value back to where the function was called.

How Return Works:

  1. Function calculates a value
  2. return sends that value back
  3. The function call becomes the returned value
  4. You can store it in a variable or use it directly
Code example 4 written in javascript. The code follows.
function add(a, b) {
  return a + b;  // Send back the sum
}

let result = add(5, 3);  // result = 8
console.log(add(10, 20));  // Prints 30

Return vs No Return:

  • Functions without return perform actions (draw, change variables)
  • Functions with return calculate and provide values
Code example 5 written in javascript. The code follows.
// Action function (no return)
function drawCircle(x, y) {
  ellipse(x, y, 50, 50);  // Draws something
}

// Value function (with return)
function calculateArea(radius) {
  return PI * radius * radius;  // Returns a number
}

let area = calculateArea(10);  // area = 314.159...

Using Returned Values:

Code example 6 written in javascript. The code follows.
function distance(x1, y1, x2, y2) {
  let dx = x2 - x1;
  let dy = y2 - y1;
  return Math.sqrt(dx * dx + dy * dy);
}

let d = distance(0, 0, 100, 100);  // d = 141.42...
if (distance(mouseX, mouseY, 200, 200) < 50) {
  // Mouse is within 50 pixels of (200, 200)
}

Important: When a function hits return, it immediately exits. Code after return won't run.

Code Organization

Functions are essential for organizing code as projects grow more complex. They break large problems into smaller, manageable pieces.

Benefits of Organization:

  • Readability: Function names describe what code does
  • Maintainability: Fix bugs or make changes in one place
  • Testing: Test individual functions separately
  • Reusability: Use functions in multiple projects
  • Collaboration: Others can understand your code structure

Before (repetitive and hard to maintain):

Code example 7 written in javascript. The code follows.
fill(255, 0, 0);
ellipse(100, 100, 50, 50);
fill(255, 0, 0);
ellipse(200, 200, 50, 50);
fill(255, 0, 0);
ellipse(300, 300, 50, 50);
// To change color, must change 3 places
// To change size, must change 3 places

After (organized with function):

Code example 8 written in javascript. The code follows.
function drawRedCircle(x, y) {
  fill(255, 0, 0);
  ellipse(x, y, 50, 50);
}

drawRedCircle(100, 100);
drawRedCircle(200, 200);
drawRedCircle(300, 300);
// To change color or size, change function once

Organizing Complex Projects:
Break your code into logical functions:

Code example 9 written in javascript. The code follows.
function setup() {
  createCanvas(400, 400);
  initializeGame();
}

function draw() {
  background(220);
  updateGame();
  drawGame();
  drawUI();
}

function initializeGame() {
  // Set up initial game state
}

function updateGame() {
  // Update game logic
}

function drawGame() {
  // Draw game elements
}

function drawUI() {
  // Draw user interface
}

Best Practices:

  • One function should do one thing well
  • Use descriptive function names
  • Keep functions focused and relatively short
  • Group related functions together

Common Patterns

Drawing Function

Code example 10 written in javascript. The code follows.
function drawFlower(x, y) {
  fill(255, 200, 0);
  ellipse(x, y, 50, 50);  // Center
  fill(255, 0, 0);
  ellipse(x - 20, y, 20, 20);  // Petal
  ellipse(x + 20, y, 20, 20);  // Petal
  ellipse(x, y - 20, 20, 20);  // Petal
  ellipse(x, y + 20, 20, 20);  // Petal
}

drawFlower(100, 100);
drawFlower(200, 200);

Function with Return

Code example 11 written in javascript. The code follows.
function calculateDistance(x1, y1, x2, y2) {
  let dx = x2 - x1;
  let dy = y2 - y1;
  return Math.sqrt(dx * dx + dy * dy);
}

let dist = calculateDistance(0, 0, 100, 100);

Modular Pattern System

Code example 12 written in javascript. The code follows.
function drawPattern(x, y, size, color) {
  fill(color);
  for (let i = 0; i < 5; i++) {
    ellipse(x + i * 10, y, size, size);
  }
}

drawPattern(50, 50, 20, color(255, 0, 0));
drawPattern(50, 150, 30, color(0, 255, 0));
