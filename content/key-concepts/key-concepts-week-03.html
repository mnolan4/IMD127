Key Concepts

Interrupts vs. Polling

p5.js gives you two ways to react to input: interrupt-style event handlers (functions that run when something happens) and polling (checking variables every frame in draw()). Knowing the difference helps you choose the right one.

Interrupt (event handler): When the user does something (e.g. clicks, presses a key), the system interrupts the normal flow and calls your function once. You define functions like mousePressed(), keyPressed(); p5.js calls them when the event occurs. Use for "when the user does X, do Y once" (e.g. change color on click, reset on key press).

Polling: Your code repeatedly checks a variable or function inside draw() every frame (e.g. mouseIsPressed, keyIsDown()). Use for "while the user is doing X, do Y every frame" (e.g. draw while mouse is held, move while arrow key is held).

Why it matters: Use interrupt handlers for one-time reactions; use polling for continuous state (held vs. not held). Picking the right one keeps your code clear and matches the interaction.

Mouse Input

p5.js provides built-in variables and functions to track mouse position and button states. These update automatically every frame, making it easy to create interactive experiences.

Mouse Position Variables:

  • mouseX: Current X coordinate of the mouse (0 to canvas width)
  • mouseY: Current Y coordinate of the mouse (0 to canvas height)
  • These update continuously as you move the mouse
Code example 1 written in javascript. The code follows.
function draw() {
  background(220);
  ellipse(mouseX, mouseY, 50, 50);  // Circle follows mouse
}

Mouse Button State:

  • mouseIsPressed: Boolean variable that is true when mouse button is held down, false when released
  • Use this in draw() to check if mouse is currently pressed (this is polling)
Code example 2 written in javascript. The code follows.
function draw() {
  if (mouseIsPressed) {
    fill(255, 0, 0);  // Red when mouse is down
  } else {
    fill(0, 0, 255);  // Blue when mouse is up
  }
  ellipse(mouseX, mouseY, 50, 50);
}

Mouse Event Functions:

  • mousePressed(): Called once when mouse button is first pressed down
  • mouseReleased(): Called once when mouse button is released
  • mouseClicked(): Called once when mouse is pressed and released (complete click)
  • These are interrupt-style event handlers: p5.js calls them automatically when the event occurs—you just define them
Code example 3 written in javascript. The code follows.
function mousePressed() {
  // This runs once when mouse button is pressed
  fill(random(255), random(255), random(255));
}

function mouseReleased() {
  // This runs once when mouse button is released
  console.log("Mouse released at", mouseX, mouseY);
}

Keyboard Input

Keyboard input allows users to control your sketch with keys. p5.js provides variables and functions to detect which keys are pressed.

Key Event Function:

  • keyPressed(): Called once when any key is first pressed down
  • keyReleased(): Called once when a key is released
  • These are interrupt-style event handlers: p5.js calls them automatically when the event occurs

Key Variables:

  • key: Contains the character of the key that was pressed (as a string)
    • Letters: 'a', 'b', 'A', 'B'
    • Numbers: '1', '2', '3'
    • Special characters: ' ' (space), 'Enter'
  • keyCode: Contains special keys that don't have character representations
    • Arrow keys: UP_ARROW, DOWN_ARROW, LEFT_ARROW, RIGHT_ARROW
    • Other: ENTER, BACKSPACE, TAB, SHIFT, CONTROL, ALT
Code example 4 written in javascript. The code follows.
function keyPressed() {
  if (key === 'r' || key === 'R') {
    // Reset when 'r' is pressed
    background(220);
  }
  
  if (keyCode === UP_ARROW) {
    // Move up when up arrow is pressed
    y -= 10;
  }
}

Continuous Key Detection:
Use keyIsDown() to check if a key is currently being held down (works in draw()). This is polling—you check the state every frame.

Code example 5 written in javascript. The code follows.
function draw() {
  if (keyIsDown(LEFT_ARROW)) {
    x -= 5;  // Move left continuously while arrow is held
  }
  if (keyIsDown(RIGHT_ARROW)) {
    x += 5;  // Move right continuously while arrow is held
  }
  ellipse(x, y, 50, 50);
}

Conditional Statements (if/else)

Conditional statements make decisions in your code based on conditions.

Basic syntax:

Code example 6 written in javascript. The code follows.
if (condition) {
  // Do this if condition is true
} else {
  // Do this if condition is false
}

Examples:

Code example 7 written in javascript. The code follows.
// Change color based on mouse position
if (mouseX < width/2) {
  fill(255, 0, 0);  // Red on left side
} else {
  fill(0, 0, 255);  // Blue on right side
}
ellipse(mouseX, mouseY, 50, 50);

Event-Driven Programming

Event-driven programming means your code responds to specific user actions (events) rather than just running continuously. When an event occurs, p5.js automatically calls the corresponding function you've defined.

How It Works:

  1. You define event functions (like mousePressed(), keyPressed())
  2. p5.js automatically calls these functions when events occur
  3. Your code inside these functions runs in response to the event

Common Events:

  • Mouse events: mousePressed(), mouseReleased(), mouseClicked(), mouseMoved()
  • Keyboard events: keyPressed(), keyReleased(), keyTyped()
  • Window events: windowResized() (when browser window is resized)

Event vs Continuous:

Event functions are interrupt-style (called when the event happens); checking mouseIsPressed or keyIsDown() in draw() is polling (you check the state every frame).

  • Event functions (mousePressed(), keyPressed()): Run once when event occurs (interrupt)
  • Continuous checking (mouseIsPressed, keyIsDown()): Check state every frame in draw() (polling)
Code example 8 written in javascript. The code follows.
// Event-driven: runs once per click
function mousePressed() {
  console.log("Clicked!");
}

// Continuous: checks every frame
function draw() {
  if (mouseIsPressed) {
    // Do something while mouse is held
  }
}

Best Practices:

  • Use event functions for one-time actions (like changing a color on click)
  • Use continuous checking in draw() for ongoing behaviors (like drawing while mouse is held)
  • Keep event functions simple and fast

Common Patterns

Mouse Following

Code example 9 written in javascript. The code follows.
function draw() {
  background(220);
  ellipse(mouseX, mouseY, 50, 50);
}

Drawing Tool

A simple drawing tool that leaves marks where you click. The key is NOT calling background() in draw(), so previous marks remain visible.

Code example 10 written in javascript. The code follows.
function setup() {
  createCanvas(400, 400);
  background(220);  // Set background once in setup
}

function draw() {
  // No background() here - this preserves previous drawings
}

function mousePressed() {
  fill(random(255), random(255), random(255));
  ellipse(mouseX, mouseY, 20, 20);
}

Variation: Drawing while dragging:

Code example 11 written in javascript. The code follows.
function setup() {
  createCanvas(400, 400);
  background(220);
}

function draw() {
  if (mouseIsPressed) {
    // Draw continuously while mouse is held
    fill(0);
    ellipse(mouseX, mouseY, 10, 10);
  }
}

Keyboard Control

Control object movement with keyboard keys. This example uses WASD keys for movement.

Code example 13 written in javascript. The code follows.
let x = 200;
let y = 200;
let moveSpeed = 5;

function draw() {
  background(220);
  ellipse(x, y, 50, 50);
}

function keyPressed() {
  if (key === 'w' || key === 'W') {
    y -= moveSpeed;  // Move up
  } else if (key === 's' || key === 'S') {
    y += moveSpeed;  // Move down
  } else if (key === 'a' || key === 'A') {
    x -= moveSpeed;  // Move left
  } else if (key === 'd' || key === 'D') {
    x += moveSpeed;  // Move right
  }
  
  // Keep object on canvas
  x = constrain(x, 25, width - 25);
  y = constrain(y, 25, height - 25);
}

Using Arrow Keys:

Code example 14 written in javascript. The code follows.
function keyPressed() {
  if (keyCode === UP_ARROW) {
    y -= 10;
  } else if (keyCode === DOWN_ARROW) {
    y += 10;
  } else if (keyCode === LEFT_ARROW) {
    x -= 10;
  } else if (keyCode === RIGHT_ARROW) {
    x += 10;
  }
}

Continuous Movement (while key held):

Code example 15 written in javascript. The code follows.
function draw() {

    x -= 5;
  }
  if (keyIsDown(RIGHT_ARROW)) {
    x += 5;
  }

    y -= 5;
  }
  if (keyIsDown(DOWN_ARROW)) {
    y += 5;

Major Project 1: Static Image, Visual Poetry

The tangible output for this week is your own visual poetry sketch created in openprocessing using the setup() static draw OR a generative system that you render a still image from. We will then print, perfect bind and have zine as a keepsake for the class. You will create the page however you would like as an artistic or poetic piece. We would love to see visual poetry experiments that explore the relationship between text and image, form and meaning, code and expression.

You can create image and/or text, but we encourage you to experiment with visual poetry—exploring how words, shapes, colors, and composition can create meaning beyond literal text. Think about how code can generate poetic visual experiences.

Pro tip: An 8.5 x 11 page could have a number of different size canvases, depending on how many Dots Per Inch you want. Higher DPI uses more power to process, lower DPI is more efficient, but lower resolution for print.

  • High-quality screen / decent print
    150 DPI → 1275 × 1650 px
    This is a sweet spot if students might export images.
  • Print-ready
    300 DPI → 2550 × 3300 px
    This is standard for professional printing, but heavy for beginners and slower in the browser.
  • Recommendation for a creative coding class
    Start with 816 × 1056 px (96 DPI).
    It maps cleanly to inches, feels "page-like," and keeps performance friendly.

Note: In addition to using a key command to export an image, there are multiple other workarounds. In the p5.js web editor, you can also take a screenshot of the preview window, right click to print frame, or for whole page file print → PDF.
