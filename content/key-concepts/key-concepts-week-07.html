Week 7: Objects and Classes - Key Concepts

Learning Outcomes

By the end of this week, you should be able to understand object-oriented programming basics. You should create classes to define object templates. You should instantiate multiple objects from a class. You should write methods that define object behaviors. You should design object-based systems with interactions.

Key Concepts

Objects

Objects are collections of related data stored as properties. They let you group variables together that belong to the same thing, making your code more organized and easier to understand.

Why Use Objects:

  • Organization: Group related data together (position, size, color for one shape)
  • Clarity: myCircle.x is clearer than circleX
  • Flexibility: Easy to add new properties
  • Multiple objects: Create many objects with the same structure

Creating Objects (Object Literals):
Objects use curly braces {} and store data as key-value pairs:

Code example 1 written in javascript. The code follows.
let myCircle = {
  x: 200,           // Property: value
  y: 200,
  diameter: 50,
  moveSpeed: 2
};

// Access properties with dot notation
console.log(myCircle.x);         // 200
console.log(myCircle.diameter);  // 50

// Modify properties
myCircle.x += myCircle.moveSpeed;
myCircle.diameter = 100;

Using Objects:

Code example 2 written in javascript. The code follows.
let myCircle = {
  x: 200,
  y: 200,
  diameter: 50,
  moveSpeed: 2
};

function draw() {
  background(220);
  myCircle.x += myCircle.moveSpeed;  // Move circle
  ellipse(myCircle.x, myCircle.y, myCircle.diameter, myCircle.diameter);
}

Multiple Objects:
Create multiple objects with the same structure:

Code example 3 written in javascript. The code follows.
let circle1 = { x: 100, y: 100, diameter: 50 };
let circle2 = { x: 200, y: 200, diameter: 75 };
let circle3 = { x: 300, y: 300, diameter: 100 };

function draw() {
  background(220);
  ellipse(circle1.x, circle1.y, circle1.diameter, circle1.diameter);
  ellipse(circle2.x, circle2.y, circle2.diameter, circle2.diameter);
  ellipse(circle3.x, circle3.y, circle3.diameter, circle3.diameter);
}

Benefits:

  • All data for one object is together
  • Easy to pass objects to functions
  • Clear what properties belong to what
  • Foundation for classes (next concept)

Classes

Classes are blueprints for creating objects. They define what properties and methods (functions) objects of that type will have. Once you define a class, you can create many instances (objects) from it.

Why Use Classes:

  • Templates: Define structure once, create many objects
  • Consistency: All instances have the same properties and methods
  • Organization: Group data and behavior together
  • Efficiency: Write code once, reuse many times

Defining a Class:

Code example 4 written in javascript. The code follows.
class Circle {
  // Constructor: runs when object is created
  constructor(x, y, diameter) {
    this.x = x;      // 'this' refers to the object being created
    this.y = y;
    this.diameter = diameter;
  }

  // Method: function that belongs to the class
  display() {
    ellipse(this.x, this.y, this.diameter, this.diameter);
  }

  // Method: can modify object properties
  move() {
    this.x += 1;
  }
}

Key Concepts:

  • class Circle: Defines the class name
  • constructor(): Special method that runs when object is created
  • this: Refers to the current object instance
  • Methods: Functions that belong to the class

Creating Instances:
Use the new keyword to create objects from a class:

Code example 5 written in javascript. The code follows.
// Create instances (objects) from the class
let circle1 = new Circle(100, 100, 50);  // x=100, y=100, diameter=50
let circle2 = new Circle(200, 200, 75);  // x=200, y=200, diameter=75

function draw() {
  background(220);
  circle1.display();  // Call method on circle1
  circle1.move();     // Move circle1
  circle2.display();  // Call method on circle2
  circle2.move();     // Move circle2
}

How this Works:
this refers to the specific object instance:

  • In circle1, this refers to circle1
  • In circle2, this refers to circle2
  • Each object has its own copy of properties

Class vs Instance:

  • Class: The template/blueprint (class Circle)
  • Instance: A specific object created from the class (circle1, circle2)
  • One class can create many instances

Methods

Methods are functions that belong to a class. They define what objects of that class can do. Methods can access and modify the object's properties using this.

Method Types:

Display Methods: Show the object visually

Code example 6 written in javascript. The code follows.
display() {
  ellipse(this.x, this.y, this.diameter, this.diameter);
}

Update Methods: Change object state over time

Code example 7 written in javascript. The code follows.
update() {
  this.x += this.speedX;
  this.y += this.speedY;
}

Behavior Methods: Implement object logic

Code example 8 written in javascript. The code follows.
bounce() {
  if (this.x > width || this.x < 0) {
    this.speedX *= -1;  // Reverse horizontal direction
  }
  if (this.y > height || this.y < 0) {
    this.speedY *= -1;  // Reverse vertical direction
  }
}

Complete Example:

Code example 9 written in javascript. The code follows.
class Ball {
  constructor(x, y) {
    this.x = x;
    this.y = y;
    this.speedX = 2;
    this.speedY = 2;
    this.diameter = 30;
  }

  update() {
    // Update position
    this.x += this.speedX;
    this.y += this.speedY;
  }

  display() {
    // Draw the ball
    ellipse(this.x, this.y, this.diameter, this.diameter);
  }
  
  bounce() {
    // Bounce off edges
    if (this.x > width || this.x < 0) {
      this.speedX *= -1;
    }
    if (this.y > height || this.y < 0) {
      this.speedY *= -1;
    }
  }
}

// Usage
let ball = new Ball(200, 200);

function draw() {
  background(220);
  ball.update();   // Update position
  ball.bounce();   // Check for bouncing
  ball.display();  // Draw the ball
}

Method Benefits:

  • Organize object behavior
  • Encapsulate logic with data
  • Reusable across all instances
  • Clear what each object can do

Common Patterns

Multiple Objects from Class

Code example 10 written in javascript. The code follows.
let circles = [];

function setup() {
  for (let i = 0; i < 10; i++) {
    circles.push(new Circle(random(width), random(height), random(20, 50)));
  }
}

function draw() {
  background(220);
  for (let circle of circles) {
    circle.update();
    circle.display();
  }
}

Object Interactions

Code example 11 written in javascript. The code follows.
class Particle {
  constructor(x, y) {
    this.x = x;
    this.y = y;
    this.vx = random(-2, 2);
    this.vy = random(-2, 2);
  }
  
  update() {
    this.x += this.vx;
    this.y += this.vy;
  }
  
  display() {
    ellipse(this.x, this.y, 10, 10);
  }
  
  checkCollision(other) {
    let d = dist(this.x, this.y, other.x, other.y);
    if (d < 20) {
      // Collision detected
      this.vx *= -1;
      this.vy *= -1;
    }
  }
}
