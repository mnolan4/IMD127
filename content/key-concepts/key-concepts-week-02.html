Key Concepts

Variables

Variables are containers that store values. They let you give names to values, making your code more readable and flexible. Instead of hard-coding numbers throughout your code, you can store them in variables and reuse or modify them easily.

Why Use Variables:

  • Flexibility: Change a value once, and it updates everywhere it's used
  • Readability: Names like x and y are clearer than random numbers
  • Reusability: Use the same value multiple times without repeating it
  • Dynamic behavior: Change values over time to create animation

Syntax:

Code example 1 written in javascript. The code follows.
let x = 200;        // Declare variable with 'let'
let y = 200;
let diameter = 100;
ellipse(x, y, diameter, diameter);  // Use variable in function call

Variable Declaration:

  • Use let to declare a variable that can be changed later
  • Use const to declare a constant value that cannot be changed
  • Variable names should be descriptive: circleX is better than x1
Code example 2 written in javascript. The code follows.
let circleX = 200;      // Can be changed
const CANVAS_SIZE = 400; // Cannot be changed (convention: UPPERCASE for constants)
circleX = 300;          // OK - variable can change
// CANVAS_SIZE = 500;   // ERROR - constant cannot change

Variable Scope:
Variables declared at the top of your file (outside functions) are available everywhere. Variables declared inside functions are only available within that function.

What the Heck is let?

You might be wondering: why do we use the word let? It's actually a normal English word that means "allow" or "permit." When you write let x = 100, you're literally saying to the computer: "Let x be 100" or "Allow x to be 100." It's like giving permission for a variable to exist and have a value.

Think of it like this: when you say "Let me explain" in conversation, you're asking for permission to explain something. In code, let is you giving the computer permission to create a variable with a specific name and value.

Animation Basics

The draw function runs continuously, creating animation.

Using frameCount:

Code example 5 written in javascript. The code follows.
function draw() {
  background(220);
  let x = frameCount;  // frameCount increases each frame
  ellipse(x, 200, 50, 50);
}

Moving shapes:

Code example 6 written in javascript. The code follows.
let x = 0;

function draw() {
  background(220);
  ellipse(x, 200, 50, 50);
  x = x + 1;  // Move right each frame
}

The Draw Loop

The draw() function is the heart of animation in p5.js. It runs continuously, approximately 60 times per second (60 frames per second, or 60 FPS). Each time draw() runs, it executes all the code inside it from top to bottom, then immediately starts again.

How Animation Works:

  1. draw() runs and draws everything
  2. A tiny amount of time passes (about 1/60th of a second)
  3. draw() runs again, drawing everything again
  4. If something has changed (like a position variable), it appears in a new location
  5. This creates the illusion of smooth movement

Important: Always Clear the Canvas
You must call background() at the start of draw() to clear the previous frame. Without it, shapes from previous frames remain visible, creating trails or overlapping shapes.

Code example 7 written in javascript. The code follows.
function draw() {
  background(220);  // Clear previous frame - CRITICAL!
  ellipse(x, y, 50, 50);
  x += 1;  // Move circle
}

Frame Rate:
The default frame rate is 60 FPS. You can change it with frameRate():

Code example 8 written in javascript. The code follows.
frameRate(30);  // Slower animation (30 FPS)
frameRate(120); // Faster animation (120 FPS, if your computer can handle it)

Frame Count:
The built-in variable frameCount automatically counts how many times draw() has run. It starts at 0 and increases by 1 each frame. This is useful for timing and creating patterns.

Common Patterns

Pulsing Animation

Code example 9 written in javascript. The code follows.
let circleSize = 50;

function draw() {
  background(220);
  circleSize = circleSize + 1;
  if (circleSize > 100) {
    circleSize = 50;
  }
  ellipse(200, 200, circleSize, circleSize);
}

Color Changing Animation

Code example 10 written in javascript. The code follows.
let redValue = 0;

function draw() {
  background(220);
  fill(redValue, 0, 0);
  ellipse(200, 200, 100, 100);
  redValue = redValue + 1;
  if (redValue > 255) {
    redValue = 0;
  }
}

Using Canvas Dimensions

Code example 11 written in javascript. The code follows.
function draw() {
  background(220);
  ellipse(width/2, height/2, 100, 100);  // Center of canvas
}

Loading External Files

Loading Images

You can load image files into your p5.js sketch to use them in your artwork. This allows you to incorporate photographs, illustrations, or any image files into your creative coding projects.

Key Steps:

  1. Declare a variable to store the image: let myImage;
  2. Load the image in preload(): myImage = loadImage('filename.jpg');
  3. Display the image using image(): image(myImage, x, y);

Why preload()?
The preload() function runs before setup() and ensures that all external files (images, sounds, data) are fully loaded before your sketch starts. This prevents errors from trying to use files before they're ready.

Code example 12a written in javascript. The code follows.
let myImage;

function preload() {
  myImage = loadImage('your-image.jpg');
}

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  image(myImage, 0, 0);
}

Image Display Options:

  • image(myImage, x, y) - Display at original size at position (x, y)
  • image(myImage, x, y, width, height) - Display at specific size
  • myImage.width and myImage.height - Get image dimensions
  • myImage.resize(width, height) - Resize the image

Supported Image Formats: JPG, PNG, GIF, SVG

Loading Images in Different Environments

p5.js Web Editor (editor.p5js.org):
Place your image file in the same folder as your sketch file, or provide the full path to the image file. Use the filename directly: loadImage('your-image.jpg')

OpenProcessing.org:
OpenProcessing uses an Assets system instead of file folders. To load images:

  1. Write your sketch code first (including the loadImage() line with your filename)
  2. Save your sketch (File > Save or Ctrl+S / Cmd+S) - this is required before uploading files
  3. Click the "Assets" button in the top toolbar (folder icon)
  4. Click "Upload" and select your image file
  5. Once uploaded, the image appears in your Assets panel
  6. Use the filename exactly as it appears in the Assets panel: loadImage('my-photo.png')

Important: You must save your sketch before the Assets panel will work properly. Write your code, save it, then upload your image file.

External URLs (works everywhere):
You can load images from the internet using a full URL: loadImage('https://example.com/path/to/image.jpg')

Exporting Your Work

Saving PNG Images

Use the save() function to export your canvas as a PNG image. This captures the current state of your canvas.

Code example 12 written in javascript. The code follows.
function keyPressed() {
  if (key === 's' || key === 'S') {
    save('my-sketch.png');  // Saves when 's' key is pressed
  }
}

Note: In the p5.js web editor, you can also take a screenshot of the preview window or use File > Export to download your sketch files.

Tips for Success

Start with static compositions before adding animation. Use variables to make your code flexible. Experiment with different animation speeds. Use width and height to make your code work with any canvas size. Always call background() at the start of draw() to clear previous frames. Use frameCount for timing and patterns. Test your animations at different frame rates to see how speed affects the visual result.
